Final Year Project (FYP) — FixMyRP (RP Campus Care)

FixMyRP is a facilities reporting app designed for Republic Polytechnic students and staff to quickly report faulty campus facilities with photos and descriptions, track report progress, and receive status update notifications.

UI/UX React Native MongoDB Reporting + Tracking Admin Panel

Project Overview

Business Issue

Students and staff often encounter faulty facilities on campus with no easy way to report them. Existing reporting methods (emails/verbal) are inefficient and lack transparency, so issues can be delayed or go unresolved.

What FixMyRP Does

  • Submit facility reports with a photo + description
  • Track report status from submission to resolution
  • Receive status updates via notifications
  • Admins can filter and update reports for faster handling

What Sets FixMyRP Apart

  • Tailored for internal school use (not public infrastructure / emergencies)
  • Custom report categories like Canteen, TRCC, Library, etc.
  • Simple student-friendly UI with minimal steps
  • Improves transparency, speed, and user communication

Product Goal

Provide an efficient and user-friendly platform for RP students and staff to report faulty facilities, streamline reporting with real-time tracking, and improve response time and communication with maintenance teams.

Requirements

Functional Requirements

  • Secure login for students & staff
  • Submit facility reports (image, location, description)
  • View own submitted reports
  • Admin can view, filter, and update reports
  • Edit profile information (user/admin)
  • Receive notifications (e.g., status updates)
  • Submit feedback to improve the app

Non-Functional Requirements

  • Mobile-first responsive experience
  • React Native front-end + MongoDB backend
  • Secure storage of data
  • Easy navigation following good UI/UX practices

Process

1) Research

Identified the campus reporting pain points (no tracking, delays, lack of transparency) and defined what an effective reporting system needs.

2) Ideation

Created early ideas (logo sketches, moodboard), then mapped the reporting flow and feature list for student + admin use cases.

3) Design

Built low-fidelity and high-fidelity wireframes in Figma, including UI kit components and page flow.

4) Build / Test

Implemented the app (React Native + MongoDB), tested on devices via Expo Go, and iterated based on issues found.

Challenges & Improvements

Challenges Encountered

  • Android build opened inconsistently due to unsigned build restrictions
  • iOS build limited without a paid Apple Developer account
  • Early coordination challenges aligning schedules and sprint tasks

Recommendations (Next Improvements)

  • Notification toggle (opt in/out)
  • Smart location selection (auto-detect + dropdown)
  • Multilingual support
  • Image annotation (mark issues on photo)
  • Enhanced image quality for clearer identification
